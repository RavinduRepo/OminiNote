# KNOWN_ISSUES.md

Tracks issues and cross-platform gaps found during codebase audits. Keep this in sync with reality: when an item is fixed, remove it (or move it to "Fixed") instead of leaving it marked done; when new issues are found, add them here rather than only mentioning them in conversation.

## Open issues

### Cross-platform

- **macOS sandbox entitlements are incomplete.** `macos/Runner/DebugProfile.entitlements` and `Release.entitlements` enable `com.apple.security.app-sandbox` but grant no file-access entitlement (e.g. `com.apple.security.files.user-selected.read-only`). PDF/image picking and reads may silently fail in macOS release builds without this.
- **Finger drawing is not available.** By design (user-approved input map): stylus + mouse draw, touch pans/zooms. A "finger draws" toggle (Samsung Notes-style) is a possible future setting.

### Canvas v1 limitations (deliberate scope cuts — see CANVAS_SPEC.md §17/§19)

- **Eraser is whole-stroke only**; partial/pixel erase later.
- **Selection is single-page**; can't lasso across page boundaries.
- **System clipboard is text-only** for paste-in (images come in via the file picker); no copy-out of images.
- **No camera capture** for images (gallery/file picker only — camera needs a plugin + permissions).
- **Undo/redo history is in-memory** (~100 ops), cleared when the canvas closes.
- **Attachments are ignored on export** (listed in UI only).
- **S-Pen button-to-erase works while the pen is touching** (`kPrimaryStylusButton` / inverted stylus); a pure hover-button tool toggle isn't implemented.
- **Perf pass pending**: committed strokes repaint every frame (no per-page picture cache yet); fine for moderate sections, revisit for very large ones (spec §14).
- **Orphaned page files**: deleting a page removes it from `section.json` but leaves `pages/<id>.json` on disk (undo may restore it). Harmless; a GC-on-open could clean these later.

### Robustness

- **IDs are timestamp-based** (`millis + 3-digit sequence` / `micros + sequence`) — fine in practice, but not UUIDs.
- **Syncfusion licensing**: export builds on `syncfusion_flutter_pdf` under the Community License (user-approved 07/06/26). If the project outgrows eligibility, swap via the `PdfExporter` interface.

### Branding

- **`flutter_native_splash` cannot be added to this project right now.** Every version requires either an `xml` range incompatible with `syncfusion_flutter_pdf ^33.2.15`'s `xml ^7.0.1`, or a `meta` version newer than the installed Flutter SDK's bundled `meta 1.17.0` — confirmed via `flutter pub add --dry-run`, a genuine current ecosystem conflict, not a config mistake. Re-check if either package's constraints loosen in a future update.
- **No native splash screen on Windows/macOS/Linux.** Android's is hand-implemented (see CLAUDE.md's Branding section); desktop splash screens would require editing native runner code (C++/Swift), not just resource files, so it was left out of scope. Flag if you want it — it's a bigger, separate effort than the Android side was.
- **iOS and web app icons are intentionally skipped** in the `flutter_launcher_icons` config (`ios: false`, no `web` block) — consistent with iOS/web not being project priorities.

## Fixed

- ~~No editable colors, no reorder, no section groups; sidebar collapse animation had artifacts~~ (07/07/26) — the notebook contents are now a recursive tree of sections + **super-sections** (OneNote-style section groups, nested arbitrarily deep). Notebooks, sections, and groups all have **editable colors** (curated swatch picker) and **rename**; items **drag-reorder** and drag between/into groups; groups **collapse/expand**. Works on both desktop (sidebar) and mobile (home = notebook reorder+color; notebook screen = full tree) via a shared `NotebookTreeView`. The sidebar collapse/expand animation artifact is fixed (content laid out at target width inside `ClipRect`+`OverflowBox` so it clips/reveals instead of reflowing). Model round-trips (incl. nested groups + legacy `sectionIds` upgrade) covered in `test/models_test.dart`. **Note:** cross-group drag covers reorder-within-a-parent and drop-into-a-group; moving an item *out* to a shallower level via drag isn't a dedicated gesture — use "Ungroup" or drop onto a higher target. Flag if you want an explicit outdent.
- ~~App had no custom icon or splash screen~~ (07/07/26) — "Stroke Page" mark (the approved Direction A logo) rasterized via `dart:ui` directly and wired up as the real launcher icon on Android (adaptive), Windows, and macOS via `flutter_launcher_icons`. Android's native splash screen (pre- and post-Android-12) was hand-implemented since `flutter_native_splash` can't currently be added (see Branding section above) — uses the app's actual light/dark canvas colors so there's no flash handing off to the first Flutter frame. Verified by a full uninstall+reinstall on two different physical Android devices (a phone and a tablet) plus a Windows build.
- ~~`DesktopShellScreen`'s sidebar could overflow its `Row` by ~94px~~ (07/07/26) — found via the above verification pass on a newly-connected tablet: the sidebar's `AnimatedContainer` used a fixed width (300 expanded / 56 collapsed) with no upper bound, so a transient near-zero-width layout frame (a known Android startup quirk — the engine briefly reports "Width is zero" before the real window size arrives) caused the header's `Row` to be laid out in far less space than its content needed. Fixed by wrapping the shell's body in a `LayoutBuilder` and clamping the sidebar's width to `constraints.maxWidth`, so it can never request more room than is actually available regardless of the cause. Confirmed clean (no overflow) on a rebuild.

- ~~No full-screen mode; toolbar options (colors/size/etc.) were always visible instead of on demand~~ (07/06/26) — added a full-screen toggle (app-bar button) that hides the app bar and toolbar entirely, replaced by two floating controls: an exit button, and a tool control collapsed to the active tool's icon that expands into the 5-tool picker on tap and shrinks back on an outside tap. Separately (and applying to **both** normal and full-screen layouts, on both mobile and desktop): each tool's options panel (colors+size for pen/highlighter, selection actions for lasso, style controls for text) is now hidden by default and only opens when you tap the *already-active* tool again (`CanvasController.setTool` toggles `toolOptionsOpen`; switching to a different tool always closes it) — tapping outside dismisses it without also drawing/panning on that tap. **Exception, by design:** panels reflecting something already in progress — actively editing text, a text box selected via lasso, or an active lasso selection/clipboard content — stay visible regardless of the toggle, since hiding those would hide the thing you're actively doing; flag if you wanted those gated too. Covered by new tests in `test/canvas_controller_test.dart`.
- ~~No desktop-appropriate layout; Notebooks and Sections views looked nearly identical~~ (07/06/26) — added a responsive shell: `_RootRouter` (in `main.dart`) picks the existing mobile push-navigation flow or a new `DesktopShellScreen` (OneNote-desktop-style: resizable notebook/section tree sidebar + the canvas embedded directly in the main pane, no pushed route) based on window width (≥840px) or a manual System/Mobile/Desktop override in Settings → Layout. Solves the "can't tell notebooks from sections" complaint too, since they're now permanently visible as distinct, nested, always-on-screen panes rather than two near-identical full-screen lists. **Caveat:** the desktop shell's bottom sheets (Add/Page settings/Attachments/Pages navigator inside the canvas) are unchanged — still sheets, not side panels; converting those to a more desktop-native form is optional future polish, not done here.
- ~~Background patterns weren't exported; changing a page's background/pattern only affected new pages, not existing ones~~ (07/06/26) — (1) `PdfExporter` now draws ruled/grid/dotted patterns as vector lines/dots on blank pages (`_drawPattern` in `pdf_exporter.dart`), mirroring `CanvasPainter._paintPattern`'s exact geometry (26pt spacing) so the export matches what's on screen; PDF-backed pages still skip it since the imported page image fully covers them, same as on screen. (2) The "Page settings" sheet's checkbox (now labeled "Apply to all pages") changes **every page currently in the canvas**, not just the section default for future pages — `CanvasController.setPageBackground(..., asSectionDefault: true)` snapshots every page's prior background so undo restores each one individually, not to one shared value. Covered by `test/canvas_controller_test.dart` and a pattern-export smoke test in `test/pdf_export_test.dart`.
- ~~PDF export: page sizes didn't match the canvas, background/design missing, pages cut off with long white gaps (worse with wide/landscape PDFs or horizontally-merged rows)~~ (07/06/26) — root cause: `sf.PdfPageSettings.size`'s setter re-derives width/height as `(min, max)` of the page's *current* orientation (default `portrait`), silently swapping any page whose intended width exceeds its height. Every horizontally-merged row (blank or PDF) and every landscape-shaped PDF page is wider than tall, so each got flipped into a taller/narrower page than intended — our drawing code kept using the original wide coordinates, so content past the new (smaller) width was clipped off and the extra height below was blank. Same root cause explained the missing background color too (the fill rect was sized correctly but drawn onto the wrong-shaped actual page, so only the clipped portion showed). **Fix** (`lib/services/pdf_exporter.dart`): assign a **fresh** `sf.PdfPageSettings(Size(w, h))` per row (constructed with size only, no orientation argument — the constructor then stores it verbatim) instead of mutating `.size` on the existing settings object. Covered by `test/pdf_export_test.dart`, including an end-to-end test that reopens the generated bytes and checks the actual page size of a merged wide row.

- ~~Text styling was whole-box only; couldn't style a sub-range; size +/- deselected~~ (07/06/26) — text is now **rich**: `TextElement` holds styled `TextRun`s and editing uses a `RichTextController` (`lib/canvas/rich_text_controller.dart`) that carries per-character styles and renders them via `buildTextSpan`. **Select part of the text and change bold/italic/size/color/family for just that range**; applying a style preserves the selection, so repeated size nudges keep working. With a collapsed caret the change becomes the typing style; lasso-selecting a whole box still styles all of it. (PDF export still draws each text box in its base style — per-run export is parked with the other export work.)
- ~~Text boxes: fixed width / forced wrap, no move-without-lasso, box-resize changed font size, font-size control deselected + only hit new text~~ (07/06/26) — text boxes now **auto-size**: they grow to the text's natural width and wrap only at the page's right edge (`autoTextRect` in `lib/canvas/text_measure.dart`, re-measured live on each keystroke). In the text tool you **tap a box to edit and drag it to move** (no lasso; reuses the move + cross-page-reparent pipeline); lasso mode also edits on tap. **Text ignores box-resize** (`TextElement.scaleBy` is a no-op for size) and text-only selections show no resize corners — only move/rotate. Font family/size/bold/italic/align/color now route through controller setters (`setTextFontSize`, etc.) that apply to the **live editing element or the text selection without deselecting**, folding into one undo op.
- ~~Side page next to a PDF ignored the default page color; cross-page item moves broke; no rename; mobile scrollbar not touchable~~ (07/06/26) — (1) horizontal pages now always take the section's default background (not the origin PDF page's blank white); (2) dragging a selection onto another page **reparents** the elements to that page (coordinates converted to the target's local space, appended on top) instead of leaving them clipped to the source page — one undoable "Move to page" op over both pages; (3) Rename added to notebooks (home menu), sections (notebook menu), and the canvas overflow menu; (4) the scrollbar thumb is now draggable by touch too (the pan recognizer no-ops while a scrollbar drag is active).
- ~~Gaps between pages / non-uniform PDF widths / rigid scrolling / no scrollbar~~ (07/06/26) — page gap is now 0 (flush, hairline-separated); inserted PDF pages are normalized to the default page width (aspect preserved); scrolling is ticker-smoothed (1:1 touch drag + inertial fling; wheel glides toward a target; Ctrl+wheel zoom, Shift+wheel horizontal); a draggable (mouse) / indicator (touch) scrollbar is drawn on overflow.
- ~~No app-wide default page color/design~~ — Settings (outside notebooks) now has a "Default page" card (color presets + Blank/Ruled/Grid/Dotted); new sections seed their default background from it.
- ~~`flutter test` failed (counter boilerplate)~~ — replaced with real model round-trip + transform tests (`test/models_test.dart`); all pass.
- ~~No locking/atomic writes in persistence~~ — all JSON writes now go through temp-file + rename (`NotebookService._writeAtomic`); per-page files keep write scope small.
- ~~Drawing input was stylus-only (desktop mouse couldn't draw)~~ — the canvas input map now routes stylus **and** primary-button mouse to drawing; touch pans/zooms; wheel scrolls (Ctrl=zoom, Shift=horizontal).
- ~~PDF and blank pages were two separate modes~~ — replaced by the uniform Canvas (rows of pages, PDF-backed or blank) per `CANVAS_SPEC.md`; pdfrx demoted to page renderer; app owns the viewport.
- ~~S-Pen scrolled the PDF while writing~~ — moot under the new architecture: stylus events never reach a pan gesture (touch-only recognizer + palm rejection while pen is down).
- ~~Line thickness (pressure) lost on reload~~ — per-point pressure persists (`p` in stroke JSON).
- ~~No settings / theme switch~~ — Settings screen with live System/Light/Dark (persisted), plus placeholder Account/Drive rows.
- ~~UI looked dated / no theming~~ — central `AppTheme`/`AppPalette`, Slate & Amber + flat Graphite shapes, fade-through transitions, staggered lists.
- ~~"Clear page" button~~ — removed per spec §7.6 (undo/redo + eraser + selection-delete cover it).
- ~~Dead code / lint debt~~ — old `note_screen.dart`, `page_screen.dart`, `stroke.dart`, `page.dart`, painters, `sample.pdf` all deleted; `flutter analyze` clean.

## Verified working / not an issue

- Platform folders (`windows/`, `linux/`, `macos/`, `android/`) are standard `flutter create` scaffolding — no structural problems.
- `flutter devices` detects the physical phone over USB even though bare `adb` isn't on PATH — use `flutter run -d <id>`.
- Legacy pre-canvas data is backed up to `notebooks_legacy.json` on first run of the new build (verified on Windows 07/06/26); the app starts with a fresh index.
